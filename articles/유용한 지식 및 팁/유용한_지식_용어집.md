# 용어집

이 문서는 유용한 지식 및 팁 문서들에서 자주 등장하는 핵심 용어를 골라, **처음 읽는 사람도 바로 이해할 수 있게** 쉬운 말로 정리한 용어집입니다.

- 조사 대상: Bash·Git·Conda·Mamba·tmux·LLM 학습/평가·데이터셋 관련 정리 문서
- 제외 기준: 너무 일상적인 단어, 실무 이해에 직접 연결되지 않는 단어
- 목표: 실무 문서를 읽을 때 최소한의 맥락을 빠르게 잡도록 돕기

총 **100개 용어**를 정리했습니다. (업데이트: 2026-03-30)

---

## 용어집

| 번호 | 용어 | 쉬운 설명 |
| --- | --- | --- |
| 1 | CLI (Command Line Interface) | 마우스 대신 명령어를 입력해 컴퓨터를 다루는 방식입니다. |
| 2 | Shell | 사용자가 입력한 명령을 운영체제에 전달해 실행해 주는 프로그램입니다. |
| 3 | Bash | Linux/Unix에서 가장 널리 쓰이는 셸 중 하나입니다. |
| 4 | 터미널 (Terminal) | 셸을 열고 명령어를 입력하는 창(프로그램)입니다. |
| 5 | Path | 파일·폴더의 위치를 나타내는 주소 문자열입니다. |
| 6 | 절대 경로 | 루트(`/`)부터 시작해 위치를 완전히 지정하는 경로입니다. |
| 7 | 상대 경로 | 현재 작업 위치를 기준으로 표현한 경로입니다. |
| 8 | 루트 디렉터리 (`/`) | 파일시스템의 최상위 시작 지점입니다. |
| 9 | 홈 디렉터리 (`~`) | 현재 사용자 계정의 기본 작업 폴더입니다. |
| 10 | 환경 변수 | 프로그램 실행에 영향을 주는 이름-값 형태의 설정입니다. |
| 11 | `PATH` | 명령어 실행 파일을 어디서 찾을지 알려 주는 환경 변수입니다. |
| 12 | 표준 입력 (stdin) | 프로그램이 입력을 받아들이는 기본 통로입니다. |
| 13 | 표준 출력 (stdout) | 프로그램이 정상 결과를 내보내는 기본 통로입니다. |
| 14 | 표준 에러 (stderr) | 프로그램이 오류 메시지를 내보내는 별도 통로입니다. |
| 15 | 리다이렉션 (`>`, `>>`) | 명령 출력 결과를 화면 대신 파일로 보내는 기능입니다. |
| 16 | 파이프 (`|`) | 앞 명령의 출력을 뒤 명령의 입력으로 연결하는 기능입니다. |
| 17 | Exit Code | 명령 실행 성공/실패를 숫자로 나타낸 값(보통 0은 성공)입니다. |
| 18 | 권한 (Permission) | 파일·폴더를 읽기/쓰기/실행할 수 있는지 정한 규칙입니다. |
| 19 | `chmod` | 파일·폴더 권한을 변경하는 명령어입니다. |
| 20 | `chown` | 파일·폴더의 소유자/그룹을 변경하는 명령어입니다. |
| 21 | 심볼릭 링크 (symlink) | 원본 파일/폴더를 가리키는 바로가기 같은 특수 파일입니다. |
| 22 | 프로세스 (Process) | 현재 실행 중인 프로그램 인스턴스입니다. |
| 23 | PID | 각 프로세스에 부여되는 고유한 번호입니다. |
| 24 | 백그라운드 실행 | 터미널을 점유하지 않고 작업을 뒤에서 계속 실행하는 방식입니다. |
| 25 | `nohup` | 터미널이 끊겨도 프로세스가 계속 돌도록 실행하는 명령입니다. |
| 26 | `grep` | 텍스트에서 원하는 패턴이 들어간 줄을 찾아 주는 도구입니다. |
| 27 | `find` | 조건에 맞는 파일/디렉터리를 검색하는 명령어입니다. |
| 28 | `sed` | 텍스트를 규칙에 따라 치환/편집하는 스트림 편집기입니다. |
| 29 | `awk` | 열(컬럼) 기반으로 텍스트를 분석·가공할 때 쓰는 도구입니다. |
| 30 | `xargs` | 입력으로 받은 목록을 다른 명령의 인자로 넘겨 대량 처리하는 도구입니다. |
| 31 | 정규표현식 (Regex) | 문자열 패턴을 간결하게 표현하는 규칙 언어입니다. |
| 32 | Git | 코드 변경 이력을 저장·관리하는 분산 버전 관리 시스템입니다. |
| 33 | Repository (Repo) | Git으로 관리되는 프로젝트 폴더입니다. |
| 34 | Commit | 특정 시점의 변경 내용을 메시지와 함께 저장한 기록 단위입니다. |
| 35 | Branch | 독립적으로 작업을 진행할 수 있는 이력의 갈래입니다. |
| 36 | Merge | 서로 다른 브랜치의 변경 이력을 하나로 합치는 작업입니다. |
| 37 | Rebase | 커밋 기반을 다른 브랜치 위로 옮겨 이력을 정리하는 작업입니다. |
| 38 | Checkout / Switch | 현재 작업할 브랜치를 바꾸는 동작입니다. |
| 39 | Pull | 원격 변경 내용을 가져와 로컬 브랜치에 반영하는 명령입니다. |
| 40 | Fetch | 원격 변경 내용을 가져오기만 하고 자동 병합은 하지 않는 명령입니다. |
| 41 | Push | 로컬 커밋을 원격 저장소로 업로드하는 명령입니다. |
| 42 | Staging Area | 커밋하기 전에 변경 파일을 모아 두는 중간 영역입니다. |
| 43 | `git add` | 커밋 대상 파일을 스테이징 영역에 올리는 명령입니다. |
| 44 | `git status` | 변경 상태, 스테이징 상태, 브랜치 상태를 보여 주는 명령입니다. |
| 45 | `git diff` | 파일의 변경 전후 차이를 보여 주는 명령입니다. |
| 46 | Merge Conflict | 같은 부분을 다르게 수정해 자동 병합이 불가능한 충돌 상황입니다. |
| 47 | Cherry-pick | 특정 커밋 하나만 골라 다른 브랜치에 적용하는 작업입니다. |
| 48 | Stash | 작업 중인 변경사항을 임시로 숨겨 두는 기능입니다. |
| 49 | Tag | 릴리스 버전처럼 특정 커밋에 이름표를 붙이는 기능입니다. |
| 50 | Remote | GitHub 같은 원격 저장소 연결 대상입니다. |
| 51 | Fork | 원본 저장소를 내 계정으로 복사해 독립적으로 작업하는 방식입니다. |
| 52 | Pull Request (PR) | 내 변경을 원본 브랜치에 합쳐 달라고 요청하는 코드 리뷰 단위입니다. |
| 53 | Code Review | 변경 코드의 품질·안전성·일관성을 확인하는 검토 과정입니다. |
| 54 | CI (Continuous Integration) | 코드 변경마다 자동 빌드·테스트를 수행하는 개발 방식입니다. |
| 55 | CD (Continuous Delivery/Deployment) | 검증된 결과물을 자동 배포까지 연결하는 개발 운영 방식입니다. |
| 56 | YAML | 설정 파일에 자주 쓰이는 사람이 읽기 쉬운 데이터 형식입니다. |
| 57 | Docker | 앱 실행 환경을 이미지로 묶어 어디서나 동일하게 실행하게 해 주는 도구입니다. |
| 58 | Container | 격리된 실행 환경에서 애플리케이션을 동작시키는 단위입니다. |
| 59 | Image | 컨테이너 실행에 필요한 파일·설정을 패키징한 템플릿입니다. |
| 60 | Dockerfile | 도커 이미지를 어떻게 만들지 정의한 스크립트 파일입니다. |
| 61 | Volume | 컨테이너 외부에 데이터를 지속 저장하기 위한 저장소 연결 방식입니다. |
| 62 | Port Mapping | 호스트 포트와 컨테이너 포트를 연결해 외부 접근을 가능하게 하는 설정입니다. |
| 63 | Virtual Environment | 프로젝트별 패키지 충돌을 막기 위해 의존성을 분리한 환경입니다. |
| 64 | Conda | 패키지와 가상환경을 함께 관리하는 도구(생태계)입니다. |
| 65 | Mamba | Conda와 호환되면서 의존성 해결 속도를 높인 패키지 관리자입니다. |
| 66 | Dependency | 코드가 동작하기 위해 필요한 외부 라이브러리/패키지입니다. |
| 67 | Lockfile | 실제 설치 버전을 고정해 환경 재현성을 높이는 파일입니다. |
| 68 | Reproducibility | 같은 코드/설정으로 언제 실행해도 같은 결과를 얻을 수 있는 성질입니다. |
| 69 | tmux | 하나의 터미널 안에서 여러 세션·창·패널을 관리하는 도구입니다. |
| 70 | Session (tmux) | 작업 상태를 유지한 채 분리/재접속할 수 있는 tmux 작업 단위입니다. |
| 71 | Pane (tmux) | 한 창을 여러 영역으로 나눠 동시에 작업할 수 있는 화면 구획입니다. |
| 72 | SSH | 원격 서버에 안전하게 접속하기 위한 암호화 통신 프로토콜입니다. |
| 73 | SSH Key | 비밀번호 대신 공개키/개인키 쌍으로 인증하는 방식입니다. |
| 74 | API | 프로그램끼리 기능과 데이터를 주고받기 위한 인터페이스 규약입니다. |
| 75 | HTTP Status Code | 요청 처리 결과를 숫자로 알려 주는 웹 표준 응답 코드입니다. |
| 76 | REST | 자원을 URL로 표현하고 HTTP 메서드로 조작하는 API 설계 스타일입니다. |
| 77 | JSON | API 통신과 설정에 널리 쓰이는 경량 데이터 형식입니다. |

| 80 | Logging | 시스템 동작 기록을 남겨 문제 분석과 운영 추적에 쓰는 활동입니다. |
| 81 | Monitoring | 서비스 상태·지표를 지속 관찰해 이상을 빠르게 감지하는 활동입니다. |
| 82 | Alerting | 임계치 초과/오류 발생 시 운영자에게 자동으로 알리는 체계입니다. |
| 83 | Latency | 요청 후 응답이 오기까지 걸리는 시간입니다. |

| 86 | SLA | 서비스 제공자가 보장하는 품질 기준(가용성, 응답시간 등) 약속입니다. |
| 87 | Backup | 데이터 손실에 대비해 별도로 복사본을 보관하는 작업입니다. |

| 100 | Guardrail | 금지 주제·출력 형식·안전 규칙을 강제해 위험 응답을 줄이는 장치입니다. |
