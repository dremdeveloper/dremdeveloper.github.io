# 마크다운 문법 친절 가이드

마크다운(Markdown)은 **읽기 쉬운 일반 텍스트 문서**를 만들면서도,
필요할 때는 제목, 목록, 링크, 코드 블록 같은 서식을 쉽게 넣을 수 있게 해 주는 문법입니다.

이 문서는 처음 배우는 사람도 이해할 수 있도록 **기초부터 자주 쓰는 문법까지** 차근차근 설명합니다.
예시는 가능한 한 바로 복사해서 써도 되도록 구성했습니다.

---

## 1. 마크다운이 좋은 이유

마크다운은 다음 같은 상황에서 특히 많이 쓰입니다.

- README 파일 작성
- 블로그 글 초안 정리
- 회의록, 메모, 문서 정리
- 개발 문서 작성
- 질문/답변 사이트나 협업 툴에서 글 작성

장점은 간단합니다.

- 문법이 쉽습니다.
- 텍스트만으로 작성할 수 있습니다.
- GitHub, Notion, Obsidian, VS Code 등 많은 도구에서 지원합니다.

## 2. 가장 먼저 알아둘 기본 원칙

### 문단 나누기

문단은 **한 줄 이상 비우면** 나뉩니다.

예시:

```md
첫 번째 문단입니다.

두 번째 문단입니다.
```

### 줄바꿈

마크다운에서 단순히 Enter 한 번만 누르면 줄이 바뀌지 않는 경우가 있습니다.
줄바꿈을 확실히 하려면 보통 아래 둘 중 하나를 씁니다.

1. 줄 끝에 공백 두 칸 넣기
2. HTML의 `<br>` 사용하기

예시:

```md
첫 번째 줄  
두 번째 줄
```

또는

```md
첫 번째 줄<br>
두 번째 줄
```

> 참고: 플랫폼마다 줄바꿈 처리 방식이 조금 다를 수 있습니다.

## 3. 제목 만들기

제목은 `#` 기호로 만듭니다.
`#` 개수가 많아질수록 더 작은 제목입니다.

```md
# 제목 1
## 제목 2
### 제목 3
#### 제목 4
##### 제목 5
###### 제목 6
```

렌더링 예시:

# 제목 1
## 제목 2
### 제목 3
#### 제목 4
##### 제목 5
###### 제목 6

### 팁

- 보통 문서 제목은 `#` 하나를 씁니다.
- 큰 단락은 `##`, 그 아래 내용은 `###`로 정리하면 보기 좋습니다.

---

## 4. 강조 문법

### 굵게

```md
**굵은 글씨**
__굵은 글씨__
```

예시: **굵은 글씨**

### 기울임

```md
*기울임 글씨*
_기울임 글씨_
```

예시: *기울임 글씨*

### 굵게 + 기울임

```md
***굵고 기울인 글씨***
```

예시: ***굵고 기울인 글씨***

### 취소선

```md
~~취소선~~
```

예시: ~~취소선~~

## 5. 목록 만들기

### 5-1. 순서 없는 목록

`-`, `*`, `+` 중 하나를 사용할 수 있습니다.

```md
- 사과
- 바나나
- 포도
```

예시:

- 사과
- 바나나
- 포도

### 하위 목록

들여쓰기를 하면 하위 목록을 만들 수 있습니다.

```md
- 과일
  - 사과
  - 바나나
- 채소
  - 당근
  - 오이
```

예시:

- 과일
  - 사과
  - 바나나
- 채소
  - 당근
  - 오이

### 5-2. 순서 있는 목록

숫자와 점을 사용합니다.

```md
1. 첫 번째
2. 두 번째
3. 세 번째
```

예시:

1. 첫 번째
2. 두 번째
3. 세 번째

### 참고

숫자를 모두 정확히 쓰지 않아도 자동 정렬되는 경우가 많습니다.

```md
1. 첫 번째
1. 두 번째
1. 세 번째
```

### 5-3. 체크리스트

GitHub 같은 곳에서는 체크박스 목록도 자주 씁니다.

```md
- [x] 완료한 일
- [ ] 아직 할 일
```

예시:

- [x] 완료한 일
- [ ] 아직 할 일

> 참고: 체크리스트는 일반 Markdown보다 **GitHub Flavored Markdown** 같은 확장 문법에서 더 잘 지원됩니다.

## 6. 링크 넣기

기본 문법은 아래와 같습니다.

```md
[보이는 글자](https://example.com)
```

예시:

[예시 링크](https://example.com)

### 제목(툴팁) 넣기

```md
[OpenAI](https://openai.com "OpenAI 홈페이지")
```

### URL만 그대로 표시하기

```md
<https://example.com>
```

예시:

<https://example.com>

---

## 7. 이미지 넣기

이미지는 링크 문법 앞에 `!`를 붙입니다.

```md
![대체 텍스트](이미지주소)
```

예시:

```md
![고양이 사진](https://example.com/cat.png)
```

### 대체 텍스트란?

이미지가 보이지 않을 때 대신 표시되는 설명입니다.
접근성 측면에서도 중요합니다.

---

## 8. 인용문 넣기

인용문은 `>` 기호를 사용합니다.

```md
> 이것은 인용문입니다.
```

예시:

> 이것은 인용문입니다.

### 여러 줄 인용

```md
> 첫 번째 줄
> 두 번째 줄
```

### 중첩 인용

```md
> 바깥 인용
>> 안쪽 인용
```

예시:

> 바깥 인용
>> 안쪽 인용

---

## 9. 코드 표현하기

개발 문서에서 가장 많이 쓰는 문법 중 하나입니다.

### 인라인 코드

문장 안의 짧은 코드는 백틱(`) 기호로 감쌉니다.

```md
`print("Hello")` 처럼 쓰면 됩니다.
```

예시:

`print("Hello")` 처럼 쓰면 됩니다.

### 코드 블록

코드가 여러 줄일 때는 백틱 3개를 사용합니다.

````md
```python
print("Hello, world!")
```
````

예시:

```python
print("Hello, world!")
```

### 언어 이름 붙이기

백틱 3개 뒤에 언어 이름을 쓰면 문법 강조가 적용되는 경우가 많습니다.

예시:

````md
```js
console.log("Hello");
```
````

```js
console.log("Hello");
```

자주 쓰는 언어 이름 예시:

- `python`
- `js` 또는 `javascript`
- `bash`
- `json`
- `html`
- `css`
- `sql`

```md

```

또는

```md

```

예시:

---

## 11. 표 만들기

표는 처음 보면 조금 낯설지만, 구조만 익히면 어렵지 않습니다.

```md
| 이름 | 역할 | 비고 |
|---|---|---|
| 민지 | 기획 | 일정 관리 |
| 준호 | 개발 | 기능 구현 |
| 서연 | 디자인 | 화면 설계 |
```

예시:

| 이름 | 역할 | 비고 |
|---|---|---|
| 민지 | 기획 | 일정 관리 |
| 준호 | 개발 | 기능 구현 |
| 서연 | 디자인 | 화면 설계 |

### 정렬하기

콜론(`:`)으로 정렬 방향을 지정할 수 있습니다.

```md
| 왼쪽 | 가운데 | 오른쪽 |
|:---|:---:|---:|
| A | B | C |
```

예시:

| 왼쪽 | 가운데 | 오른쪽 |
|:---|:---:|---:|
| A | B | C |

- `:---` → 왼쪽 정렬
- `:---:` → 가운데 정렬
- `---:` → 오른쪽 정렬

## 12. 수평 규칙과 문단 조합 예시

문서가 길어지면 제목, 목록, 코드 블록, 표를 섞어서 씁니다.
아래는 실제 문서에서 자주 보이는 형태입니다.

````md
# 프로젝트 소개

이 프로젝트는 사용자 데이터를 분석하는 도구입니다.

## 주요 기능

- 파일 업로드
- 통계 계산
- 결과 시각화

## 사용 예시

```bash
python app.py
```
````

## 13. 이스케이프(특수문자 그대로 쓰기)

마크다운 문법 기호를 **문법이 아니라 문자 그대로** 보여 주고 싶을 때는
앞에 역슬래시 `\`를 붙입니다.

예시:

```md
\*이것은 기울임이 아니라 별표 문자입니다.
\# 이것은 제목이 아닙니다.
```

결과 예시:

\*이것은 기울임이 아니라 별표 문자입니다.  
\# 이것은 제목이 아닙니다.

## 14. HTML도 함께 쓸 수 있나요?

많은 마크다운 환경에서는 HTML 일부를 함께 사용할 수 있습니다.
예를 들어 줄바꿈, 접기, 색상 같은 기능이 Markdown만으로 부족할 때 HTML을 섞기도 합니다.

예시:

```md
<details>
  <summary>눌러서 펼치기</summary>
  숨겨진 내용입니다.
</details>
```

> 단, 모든 플랫폼이 HTML을 동일하게 지원하는 것은 아닙니다.

## 15. GitHub에서 자주 쓰는 추가 문법

일반 Markdown 외에 GitHub에서는 아래 기능도 자주 씁니다.

### 체크박스

```md
- [x] 문서 작성
- [ ] 배포하기
```

### 코드 문법 강조

언어 이름을 붙인 코드 블록이 잘 지원됩니다.

### 표

표 문법도 비교적 안정적으로 잘 보입니다.

### 멘션, 이슈 링크

GitHub에서는 아래처럼 쓸 수도 있습니다.

```md
@username
#123
```

이 기능은 GitHub 같은 특정 서비스에서만 특별하게 동작합니다.

## 16. 자주 하는 실수

### 1) 제목 뒤에 띄어쓰기를 안 함

```md
#제목
```

이렇게 쓰면 제목으로 인식되지 않는 환경이 있습니다.
보통은 아래처럼 쓰는 것이 안전합니다.

```md
# 제목
```

### 2) 목록 안에서 들여쓰기가 들쭉날쭉함

하위 목록이나 코드 블록은 들여쓰기가 어긋나면 모양이 깨질 수 있습니다.

### 3) 줄바꿈이 예상과 다름

마크다운은 워드프로세서처럼 보이는 대로 줄바꿈되지 않을 수 있습니다.
필요하면 빈 줄을 넣거나 `<br>`를 사용하세요.

### 4) 플랫폼마다 지원 문법이 다름

예를 들어 체크리스트, 표, 접기 기능은 어떤 곳에서는 잘 되고 어떤 곳에서는 제한될 수 있습니다.

## 17. 가장 자주 쓰는 문법만 빠르게 보기

| 기능 | 문법 예시 |
|---|---|
| 제목 1 | `# 제목` |
| 제목 2 | `## 제목` |
| 굵게 | `**굵게**` |
| 기울임 | `*기울임*` |
| 취소선 | `~~취소선~~` |
| 목록 | `- 항목` |
| 번호 목록 | `1. 항목` |
| 링크 | `[이름](주소)` |
| 이미지 | `![설명](주소)` |

| 인라인 코드 | `` `code` `` |
| 코드 블록 | `` ```언어 `` |
| 가로줄 | `---` |
| 표 | `| A | B |` |
| 체크박스 | `- [ ] 할 일` |

## 18. 바로 복사해서 쓸 수 있는 문서 템플릿

아래 템플릿은 README, 공부 노트, 프로젝트 문서 등에 두루 쓸 수 있습니다.

````md
# 문서 제목

문서를 한 줄로 소개합니다.

## 개요

- 이 문서의 목적
- 필요한 배경 설명
- 핵심 내용 요약

## 사용 방법

1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계

## 예시

```bash
example command
```

## 참고 링크

- [공식 문서](https://example.com)
- [관련 자료](https://example.com)

## 할 일

- [x] 초안 작성
- [ ] 내용 보완
- [ ] 최종 검토
````

## 19. 초보자에게 드리는 추천 방법

처음부터 모든 문법을 외우려고 하지 않아도 됩니다.
아래 순서만 익혀도 대부분의 문서를 충분히 작성할 수 있습니다.

1. 제목
2. 목록
3. 굵게
4. 링크
5. 코드 블록
6. 표

이 여섯 가지만 익혀도 실사용에 큰 문제가 없습니다.

## 20. 마무리

마크다운은 **간단하지만 강력한 문서 작성 도구**입니다.
처음에는 기호가 낯설 수 있지만, 몇 번만 써 보면 금방 익숙해집니다.

가장 좋은 연습 방법은 아래와 같습니다.

- 짧은 메모를 마크다운으로 써 보기
- README 파일 흉내 내 보기
- 표, 링크, 코드 블록을 직접 한 번씩 작성해 보기

문법을 모두 외우기보다,
**자주 쓰는 문법부터 직접 써 보면서 익히는 것**이 가장 빠릅니다.

---

## 부록: 예제 한 장으로 보기

````md
# 오늘의 공부 정리

## 배운 내용

- **마크다운 제목** 만들기
- *기울임* 과 **굵게** 사용하기
- [링크](https://example.com) 넣기
- 코드 작성하기

## 예제 코드

```python
def hello():
    print("Hello, Markdown!")
```

## 할 일

- [x] 기본 문법 보기
- [ ] 표 문법 연습하기
- [ ] README 작성해 보기
````

이 정도만 익혀도 기본 문서는 충분히 만들 수 있습니다.
